# 重建完整性检查表

## 重建等级

- L1 功能说明：只能理解用途。
- L2 接口重建：可恢复路径、类型和公开接口。
- L3 行为重建：可恢复流程、状态、错误处理和可观察行为。
- L4 结构重建：进一步恢复私有成员、初值、内部方法、算法步骤和调用顺序。

默认目标为 L4；视觉资源或生成文件可按实际价值降级并说明。

## 检查项

- [ ] 所有拥有文件都有准确路径。
- [ ] 技术栈、版本约束和外部依赖明确。
- [ ] 类型、常量、默认值和单位齐全。
- [ ] 公开签名和兼容契约齐全。
- [ ] 私有状态、初值和生命周期齐全。
- [ ] 初始化、更新、销毁顺序明确。
- [ ] 算法能按步骤实现，不依赖“按原逻辑”等模糊表达。
- [ ] 状态、事件、守卫、动作和下一状态齐全。
- [ ] 异步、并发、超时和重试语义明确。
- [ ] 输入输出、协议、持久化 schema 明确。
- [ ] 无效输入、失败和恢复路径明确。
- [ ] 日志和外部可观察行为明确。
- [ ] 安全约束和不变量明确。
- [ ] 验收条件可执行，并区分自动与人工验证。

任何缺项都应标为信息缺口，不能通过想象补成既定需求。
