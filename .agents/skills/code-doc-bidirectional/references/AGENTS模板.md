# <项目名称>长期开发纪律

只记录长期稳定、跨模块适用的规则；具体接口、参数和行为写在模块文档。

## 开始开发前

- 阅读 README 和目标模块文档。
- 检查未提交改动并保护用户工作。

## 代码与文档同步

- 代码行为变化同步重建规格。
- 新增、合并、拆分、删除模块同步 README 导航。
- 文档驱动开发完成后反查代码一致性。

## 实现纪律

根据项目提取架构边界、安全、兼容性、敏感信息和性能纪律。不能从项目确认的规则标为待确认，不编造。

## 验证与交付

- 运行可用的构建、静态检查和测试。
- 区分自动验证与需要人工或硬件完成的验证。
- 报告行为、文档、兼容性和未验证项。
