## 总：模块概要（给人阅读）

本模块是项目的集中配置中心。硬件接线、舵机动作、人员检测、门控时间、BLE 标识和网络行为等固定参数都从这里提供，其他模块不再各自保存一份相同数值。

把参数集中管理的意义，是让“系统行为为什么发生变化”更容易追踪。例如门关得太快，可以从门控时间配置开始检查；更换传感器接线，可以从 I²C 引脚配置开始修改，而不必在多个源文件中搜索数字。

### 配置怎样影响系统

```text
配置参数
├─ 硬件层：I²C 引脚、舵机引脚、TOF 地址
├─ 动作层：开关门角度、舵机速度
├─ 判断层：距离阈值、防抖、驻留和关门延时
├─ 通信层：BLE UUID、设备名、Web 端口、mDNS
└─ 运行层：串口、调试输出、连接与重启时间
```

### 参数分组

| 分组 | 主要内容 | 主要使用者 |
|---|---|---|
| `Pins` | I²C、舵机和点动开关引脚 | 系统入口、硬件驱动 |
| `Servo` | 开关门角度、运动间隔和步长 | 舵机控制、门控 |
| `Tof` | 地址、检测阈值和有效量程 | TOF、门控 |
| `Door` | 防抖、人员驻留和关门延时 | 自动门控制 |
| `Ble` | 设备名、服务和特征 UUID | BLE 配网 |
| `Network` | mDNS、端口和网络时序 | Wi-Fi、Web 服务 |
| `Debug`、`Serial`、`System` | 日志、串口和重启时序 | 系统入口及各模块 |

### 修改前先判断影响范围

| 影响等级 | 典型参数 | 修改后要做什么 |
|---|---|---|
| 硬件安全 | 舵机角度、步长、引脚 | 核对接线，脱离门体进行台架测试 |
| 业务行为 | 检测阈值、驻留、关门延时 | 重新验证人员靠近、离开和折返场景 |
| 外部兼容 | BLE UUID、设备名、mDNS、端口 | 同步手机端、网页入口或其他客户端 |
| 调试体验 | 日志开关、打印间隔 | 检查串口负载和日志内容 |

这些配置是编译期默认值。closedAngle、openAngle 与 changeThresholdCm 作为首次运行或 NVS 参数无效时的默认值；实际初始角度、相对转动角度和距离阈值可从网页写入 `doorcfg`，无需重新编译，并在重启后继续生效。

### 3. 不变量与验收

- 时间字段后缀 `Ms` 表示毫秒，距离字段后缀 `Cm` 表示厘米。
- UUID、设备名和 mDNS 名称属于外部契约，修改需同步 BLE/Wi-Fi 文档。
- 重建时不得把 `DoorState` 移入 `Config` 命名空间，否则现有调用会失配。
- 目标开发板为带板载 USB 转串口芯片的 ESP32-C3；Arduino IDE 选择 ESP32C3 Dev Module，并关闭 `USB CDC On Boot`，串口监视器波特率为 115200。
- 所有可关闭的串口日志必须由 `Config::Debug` 集中控制，不得在模块中另设分散开关。`enabled=false` 一键关闭全部日志；`enabled=true` 时各分类开关生效。默认关闭高频周期日志、逐样本、底层事件、扫描明细、HTTP轮询和包含密码的 BLE payload；保留启动摘要、错误、TOF概要、Wi-Fi结果和 BLE生命周期。
- `logBlePayload=true` 会输出 Wi-Fi 密码，只允许在受控调试环境短时开启。
- 首次运行且 `doorcfg` 尚无保存值时，舵机默认初始/关门角为 0°，默认相对转角为 `100-0=+100°`，默认开门角为 100°。已有 NVS 参数优先于这些编译默认值。
