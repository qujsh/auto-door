# WebSocket 实时状态

- 状态：Proposed
- 关联路线图：Phase 4

## 目标

由 ESP32 主动推送状态，减少固定 REST 轮询，并为曲线和日志流提供通道。

## 非目标

- 不实现日志、曲线、认证或互联网访问。
- 不删除 `/api/status`。

## 目标行为

- 新增 `/ws`。
- 初次连接立即发送完整状态。
- 状态变化时推送，并提供低频快照或保活。
- 页面断线后退避重连，不可用时回退 REST 轮询。
- 多个局域网客户端可同时接收状态。

## 接口

JSON 字段与 `/api/status` 一致，增加 `type:"status"` 和单调递增 `sequence`。

## 验收条件

- [ ] 页面连接后立即显示第一份状态。
- [ ] 门或舵机变化后客户端收到新 sequence。
- [ ] 两个客户端同时接收更新。
- [ ] 断线后自动恢复，期间 REST 回退可用。
- [ ] `/api/status` 保持兼容。
- [ ] 失效客户端不会持续累积资源。

需完成编译、双客户端、断网恢复和 30 分钟稳定性检查。
