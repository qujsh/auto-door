# 验证手册

## 验证层级

1. 静态检查：接口、状态机、非阻塞性、敏感信息和文档一致性。
2. 编译检查：使用项目实际 ESP32 board FQBN 编译全部源文件。
3. 台架检查：舵机脱离门体，检查角度、方向和读数。
4. 整机检查：低风险环境下验证完整门体行为。

## 编译

仓库尚未记录唯一可靠的 board FQBN。补齐前必须从本机 Arduino 配置或用户信息确认，不能猜测开发板型号。确认后把命令写回本节：

```text
arduino-cli compile --fqbn <verified-fqbn> <project-path>
```

## 最低检查

- [ ] 未引入长时间阻塞。
- [ ] AUTO、MANUAL 优先级没有意外改变。
- [ ] 无效距离不会触发危险转换。
- [ ] 网络失败不阻止舵机和门控制更新。
- [ ] 日志和响应不包含 Wi-Fi 密码。
- [ ] 修改接口已同步到 `interfaces.md`。
- [ ] feature spec 每项验收条件有结果。

## 核心硬件场景

| 场景 | 预期结果 |
|---|---|
| 上电且无人 | 标定完成，门关闭 |
| 人持续接近超过防抖 | 门开始打开 |
| 人继续停留 | 门保持打开 |
| 人离开、延时未到 | 等待关门 |
| 等待时人再次出现 | 取消关门 |
| 无人且延时到期 | 门开始关闭 |
| 无效距离 | 本轮不推进状态机 |
| 切换 MANUAL | 自动状态机暂停，网页可控 |
| Wi-Fi 断开 | 自动门继续工作并重连 |

交付时明确标记：已静态检查、已编译、已台架验证、已整机验证、待硬件验证。
